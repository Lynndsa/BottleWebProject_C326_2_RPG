## О проекте

Веб-приложение реализует три задачи из различных предметных областей, каждая из которых основана на классических алгоритмах теории графов:

| Вариант | Тематика                             | Алгоритм       |
| ------- | ------------------------------------ | -------------- |
| 1       | Эпидемиология и сетевая безопасность | BFS            |
| 2       | Транспортная логистика               | Dijkstra + TSP |
| 3       | Анализ финансовых транзакций         | DFS            |

Приложение позволяет:

* вводить граф вручную;
* загружать данные из `.txt` файлов;
* генерировать тестовые данные;
* визуализировать графы в формате SVG;
* получать результаты работы алгоритмов через веб-интерфейс.

# Реализованные алгоритмы

## Моделирование распространения вируса (BFS)

### Постановка задачи

Имеется неориентированный граф G = (V, E):

* вершины (V) — люди или компьютеры;
* рёбра (E) — связи и контакты между ними.

Задано подмножество изначально заражённых вершин V_inf (первичные очаги инфекции).

На каждом дискретном шаге времени инфекция пытается распространиться на здоровых соседей текущего фронта волны с заданной вероятностью передачи p. Процесс симуляции повторяется циклически I раз для сбора усредненной стохастической статистики методом Монте-Карло.

### Используемый алгоритм

**Вероятностный обход графа в ширину (Probabilistic BFS)**

* **Препроцессинг:** Предварительный анализ графа на связность (BFS_Connectivity) от начальных очагов для определения теоретического максимума достижимых вершин X. Если граф несвязен, симуляция выдает предупреждение.
* **Симуляция:** Послойный обход графа по уровням удалённости от источника с использованием очереди (Q) и генерацией случайных чисел Rand(0..1) < p для моделирования случайного характера заражения.

## Planирование экскурсий (TSP)

### Постановка задачи

Задана дорожная сеть:

* количество вершин: `N ≤ 50`;
* имеется отель;
* задан список достопримечательностей `M ≤ 8`.

Требуется построить кратчайший маршрут:

* начинается в отеле;
* проходит через все достопримечательности;
* заканчивается в отеле.

### Используемые алгоритмы

1. **Алгоритм Дейкстры**

   * поиск кратчайших расстояний между ключевыми вершинами.

2. **Полный перебор перестановок**

   * поиск оптимального порядка посещения объектов.

---

## Анализ финансовых транзакций (DFS)

### Постановка задачи

Рассматривается ориентированный граф транзакций:

* вершины — кошельки;
* дуги — денежные переводы;
* каждая дуга содержит:

  * сумму перевода;
  * временную метку.

Необходимо обнаружить длинные цепочки последовательного перемещения средств, которые могут свидетельствовать о схемах отмывания денег.

### Используемый алгоритм

**Depth-First Search (DFS) с backtracking**

Переход по ребру выполняется только если:

```text
next_vertex ∉ visited
timestamp > previous_timestamp

```

Алгоритм ищет максимальные ациклические пути движения средств.

# Форматы входных данных

## BFS — параметры модели и рёбра графа

В управляющем интерфейсе веб-формы задаются параметры:

* `N` — Общее число вершин в популяции ($1 \le N \le 50$);
* `M` — Количество стартовых очагов инфекции ($0 < M \le N$);
* `p` — Вероятность успешной передачи инфекции ($0 < p \le 1.0$);
* `I` — Число запусков модели Монте-Карло для набора статистики;
* `V_inf` — Массив номеров стартовых зараженных вершин.

Сеть контактов передается списком рёбер:

```text
<вершина_u> <вершина_v>

1 2
2 3
1 3

```

---

## TSP — рёбра графа

```text
<вершина_u> <вершина_v> <вес>

1 2 5
2 3 3
1 3 10

```

## DFS — транзакции

```text
<отправитель> <получатель> <сумма> <время>

D4E5F6 G7H8I9 750.00 1700000120

```

# Результаты работы

После выполнения алгоритма пользователь получает:

* SVG-визуализацию графа;
* статистику по графу;
* найденные маршруты и цепочки;
* подсветку подозрительных путей;
* таблицу исходных данных;
* для модуля BFS: расчет математического ожидания длительности эпидемии ($V_{mean}$), финальный средний процент зараженной популяции ($P_{final}$) и графики зависимости количества больных узлов от дискретного времени.

Результаты отображаются непосредственно на веб-странице.
