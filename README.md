  - Bottle 0.12+          — лёгкий WSGI веб-фреймворк (один файл, без зависимостей)
  - Graphviz / svgwrite   — генерация SVG-визуализаций графов (если подключено)
  - HTML5 / CSS3 / JS     — фронтенд без сторонних фреймворков
  - Visual Studio / VS Code — среда разработки

  Установка зависимостей:
      pip install bottle

--------------------------------------------------------------------------------
  РЕШАЕМЫЕ ЗАДАЧИ
--------------------------------------------------------------------------------

  Вариант 1 — Эпидемиология и сетевая безопасность (BFS)
  -------------------------------------------------------
  Моделирование распространения инфекции или компьютерного вируса в связной
  сети. Вершины — узлы (люди / компьютеры), рёбра — физические или социальные
  связи с заданной вероятностью передачи. Алгоритм: обход в ширину (BFS)
  от начального заражённого узла с учётом вероятности передачи на каждом шаге.

  Вариант 2 — Транспортная логистика (TSP)
  -----------------------------------------
  Задача коммивояжёра на подмножестве вершин графа. Имеется дорожная сеть
  (N вершин, N ≤ 50), выделенный узел-отель и список из M достопримечательностей
  (M ≤ 8). Необходимо найти кратчайший замкнутый цикл, начинающийся и
  заканчивающийся в отеле и проходящий через все M заданных вершин.
  Алгоритм: Дейкстра (построение матрицы расстояний) + полный перебор
  перестановок (Brute Force, O(M!)).

  Вариант 3 — Анализ финансовых транзакций (DFS)
  ------------------------------------------------
  Мониторинг ориентированных графов блокчейн-переводов. Вершины — адреса
  кошельков, дуги — транзакции с временными метками и суммами. Задача:
  поиск максимальных бесциклических путей перемещения средств для выявления
  многоуровневых схем отмывания денег (слоинг). Алгоритм: DFS с backtracking,
  переход v→u разрешён только если u ∉ visited и метка дуги > last_ts.

--------------------------------------------------------------------------------
  ОСОБЕННОСТИ ЗАПУСКА
--------------------------------------------------------------------------------

  1. Убедитесь, что установлен Python 3.10+ и пакет bottle:
         pip install bottle

  2. Перейдите в корневую папку проекта (там, где находится app.py).

  3. Запустите сервер:
         python app.py

  4. Откройте браузер и перейдите по адресу:
         http://localhost:8080

  Примечания:
  - Порт и хост задаются в файле app.py (переменные HOST, PORT).
  - Режим отладки (debug=True) включён по умолчанию для разработки.
    Перед деплоем установите debug=False и reloader=False.
  - Статические файлы (CSS, изображения) раздаются встроенным роутом
    /static/<filepath> из папки ./static/.

--------------------------------------------------------------------------------
  СТРУКТУРА ФАЙЛОВ ПРОЕКТА
--------------------------------------------------------------------------------

  app.py                        — точка входа, запуск сервера Bottle
  routes.py                     — маршруты (роуты) приложения

  --- Алгоритмы ---
  bfs_algorithm.py              — алгоритм BFS (распространение вируса)
  tsp_algorithm.py              — алгоритм TSP (Дейкстра + перебор)
  dfs_algorithm.py              — алгоритм DFS (поиск путей в транзакциях)

  --- Валидаторы ---
  bfs_validator.py              — валидация входных данных для BFS
  tsp_validator.py              — валидация входных данных для TSP
  dfs_validator.py              — валидация входных данных для DFS

  --- Шаблоны (views/) ---
  views/layout.tpl              — базовый шаблон (шапка, навигация, подвал)
  views/index.tpl               — главная страница с описанием задач
  views/bfs.tpl                 — страница модуля BFS
  views/tsp.tpl                 — страница модуля TSP
  views/dfs.tpl                 — страница модуля DFS (Вариант 3)
  views/about.tpl               — страница «Об авторах»

  --- Статика (static/) ---
  static/content/site.css       — общие стили сайта
  static/content/menu.css       — стили навигации
  static/content/bfs.css        — стили страницы BFS
  static/content/excursion.css  — стили страницы TSP
  static/content/dfs.css        — стили страницы DFS
  static/favicon.ico            — иконка сайта

  --- Входные данные ---
  Пользователь вводит данные через веб-интерфейс тремя способами:
    1) Ручной ввод в текстовое поле (textarea)
    2) Случайная генерация (параметры задаются на странице)
    3) Загрузка из текстового файла .txt

  Формат входного файла для DFS (транзакции):
      <отправитель> <получатель> <сумма> <метка_времени>
      Пример:
          A1B2C3 D4E5F6 1500.00 1700000001
          D4E5F6 G7H8I9  750.00 1700000120

  Формат входного файла для TSP (рёбра графа):
      <вершина_u> <вершина_v> <вес>
      Пример:
          1 2 5
          2 3 3
          1 3 10

  --- Выходные данные ---
  Результаты отображаются непосредственно на веб-странице:
    - SVG-визуализация графа
    - Сводная статистика (кол-во вершин, рёбер, длина цепочки и т.д.)
    - Список найденных путей / маршрутов с разметкой подозрительных
    - Таблица входных данных с подсветкой

  Выходные данные не сохраняются в файл автоматически.
  При необходимости пользователь может скопировать результаты со страницы.
